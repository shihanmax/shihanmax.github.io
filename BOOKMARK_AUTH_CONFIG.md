# 书签管理系统认证配置

## 概述
为了防止他人随意修改、新增或删除书签，系统现在已经实现了基于Session的管理员认证机制。

## 功能特点
- ✅ **查看书签**: 所有用户都可以查看书签列表和搜索
- 🔒 **修改书签**: 只有登录的管理员才能添加、编辑、删除书签
- 🛡️ **会话管理**: 基于Flask Session的安全会话管理
- 🎨 **界面适配**: 根据登录状态动态显示/隐藏管理功能

## 配置步骤

### 1. 修改管理员凭据
在 `app.py` 文件中修改以下配置：

```python
# 管理员认证配置
ADMIN_USERNAME = 'admin'  # 修改为您的用户名
ADMIN_PASSWORD = 'your-password-here'  # 请修改为您的密码
```

**重要**: 请务必将 `ADMIN_PASSWORD` 修改为您自己的强密码！

### 2. 更新Secret Key
为了安全起见，请修改Flask的SECRET_KEY：

```python
app.config['SECRET_KEY'] = 'your-secret-key-here'  # 修改为随机字符串
```

## 使用方式

### 普通用户
- 可以查看所有书签
- 可以搜索书签
- 无法看到"添加书签"按钮和编辑/删除按钮
- 可以看到"管理员登录"按钮

### 管理员
1. 点击页面上的"管理员登录"按钮
2. 输入配置的用户名和密码
3. 登录成功后可以看到：
   - "添加书签"按钮
   - 每个书签的"编辑"和"删除"按钮
   - "登出"按钮

## 安全机制

### API保护
- `POST /api/bookmarks` - 添加书签（需要认证）
- `PUT /api/bookmarks/<id>` - 更新书签（需要认证）
- `DELETE /api/bookmarks/<id>` - 删除书签（需要认证）
- `GET /api/bookmarks` - 查看书签（公开访问）

### 前端保护
- 根据登录状态动态显示/隐藏管理功能
- 操作前进行权限检查
- 权限错误时自动刷新页面状态

### 会话管理
- 基于Flask Session的安全会话
- 支持登录/登出操作
- 会话状态检查API

## 访问地址
- 书签页面: http://127.0.0.1:8081/bookmarks
- 管理员登录: http://127.0.0.1:8081/admin/login
- 登出: http://127.0.0.1:8081/admin/logout

## 注意事项
1. 请妥善保管管理员凭据
2. 建议在生产环境中使用HTTPS
3. 定期更新密码和Secret Key
4. 可以考虑添加更复杂的认证机制（如双因素认证）

## 故障排除
- 如果登录后仍然看不到管理功能，请刷新页面
- 如果忘记密码，请修改 `app.py` 中的 `ADMIN_PASSWORD` 并重启应用
- 如果会话异常，请清除浏览器Cookie后重新登录
